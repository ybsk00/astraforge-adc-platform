# ADC 플랫폼 UI 디자인 요구사항 명세서

UI/UX 디자인을 위한 페이지 및 기능 명세입니다.

---

## 📊 전체 페이지 요약

| 섹션 | 페이지 수 | 설명 |
|---|:---:|---|
| 공개 페이지 | 2 | 랜딩, 로그인 |

## 1. 공개 페이지 (Public)

### 1.1 인덱스/랜딩 페이지
- **경로**: `/`
- **목적**: 서비스 소개 및 로그인 유도
- **주요 요소**:
  - 히어로 섹션 (서비스 소개)
  - 주요 기능 하이라이트
  - CTA 버튼 (로그인/시작하기)
  - 푸터 (연락처, 링크)

### 1.2 로그인 페이지
- **경로**: `/login`
- **목적**: 사용자 인증
- **주요 요소**:
  - 이메일/비밀번호 입력
  - 로그인 버튼
  - 소셜 로그인 (Google/GitHub) - 선택
  - 비밀번호 찾기 링크

## 2. 사용자 페이지 (로그인 후)

### 2.1 대시보드
- **경로**: `/dashboard`
- **목적**: 사용자 워크스페이스 홈
- **주요 요소**:
  - 최근 Design Run 목록
  - 빠른 액션 버튼
  - 통계 요약 카드

---

### 2.2 Design Run (설계 실행) - 4페이지

#### 2.2.1 런 목록
- **경로**: `/design/runs`
- **기능**:
  - 런 리스트 (테이블/카드)
  - 상태 필터 (진행중/완료/실패)
  - 새 런 생성 버튼 → 모달
  - 페이지네이션

#### 2.2.2 런 상세 + 후보 목록
- **경로**: `/design/runs/[id]`
- **기능**:
  - 런 정보 헤더 (적응증, 전략, 상태)
  - 후보 목록 테이블
  - 4축 스코어 요약 (Eng/Bio/Safety/Total)
  - 필터/정렬 옵션
  - 후보 선택 → 비교 버튼

#### 2.2.3 후보 상세
- **경로**: `/design/runs/[id]/candidates/[candidateId]`
- **기능**:
  - 후보 구성 정보 (Target/Antibody/Linker/Payload)
  - 4축 스코어 레이더 차트
  - Score Components 상세
  - Evidence 섹션 (문헌 인용)
  - Protocol 추천 섹션
  - 피드백 입력 폼

#### 2.2.4 후보 비교
- **경로**: `/design/runs/[id]/compare?ids=a,b`
- **기능**:
  - 2개 후보 병렬 비교
  - 스코어 차이 시각화
  - Evidence 비교
  - 선호도 투표 버튼

---

### 2.3 Catalog (카탈로그) - 3페이지

#### 2.3.1 카탈로그 목록
- **경로**: `/catalog`
- **기능**:
  - 탭: Target / Payload / Linker / Antibody
  - 검색 및 필터
  - 컴포넌트 카드/테이블
  - 품질 등급 배지 (Gold/Silver/Bronze)

#### 2.3.2 컴포넌트 상세
- **경로**: `/catalog/[id]`
- **기능**:
  - 기본 정보 (이름, 타입, 상태)
  - 속성 테이블 (RDKit 디스크립터)
  - 외부 ID 링크 (UniProt, ChEMBL, PubChem)
  - 관련 런 히스토리

#### 2.3.3 컴포넌트 등록/편집
- **경로**: `/catalog/new` 또는 `/catalog/[id]/edit`
- **기능**:
  - 타입 선택
  - 속성 입력 폼
  - SMILES 입력 (화합물)
  - 품질 등급 선택
  - 저장/취소 버튼

## 3. 관리자 페이지 (Admin)

### 3.1 커넥터 관리 - 2페이지

#### 3.1.1 커넥터 대시보드
- **경로**: `/admin/connectors`
- **기능**:
  - 커넥터 카드 그리드 (PubMed, UniProt, ChEMBL 등)
  - 상태 표시 (idle/running/failed)
  - 마지막 실행 시간
  - "Run" 버튼

#### 3.1.2 커넥터 상세
- **경로**: `/admin/connectors/[source]`
- **기능**:
  - 실행 이력 테이블
  - 로그 뷰어
  - 설정 편집

---

### 3.2 스테이징 승인
- **경로**: `/admin/staging`
- **기능**:
  - 대기 중인 컴포넌트 목록
  - 상세 정보 패널
  - 승인/거절 버튼
  - 일괄 승인 체크박스

---

### 3.3 수집 모니터링 - 2페이지

#### 3.3.1 수집 로그
- **경로**: `/admin/ingestion/logs`
- **기능**:
  - 로그 테이블 (시간/소스/상태/메시지)
  - 필터 (소스, 상태, 날짜)
  - 상세 보기 모달

#### 3.3.2 수집 통계
- **경로**: `/admin/ingestion/stats`
- **기능**:
  - 일별 수집량 차트
  - 소스별 통계 테이블
  - 성공률 지표

---

### 3.4 관측 대시보드
- **경로**: `/admin/observability`
- **기능**:
  - 시스템 헬스 상태
  - 일별 처리량 트렌드 차트
  - 소스별 성공률 차트
  - 최근 오류 로그

---

### 3.5 알림 관리
- **경로**: `/admin/alerts`
- **기능**:
  - 알림 목록 (오류/경고/정보)
  - 읽음/안읽음 필터
  - 읽음 처리 버튼
  - 알림 삭제

## 4. 공통 컴포넌트

### 4.1 레이아웃
- **헤더**: 로고, 네비게이션, 사용자 메뉴
- **사이드바**: 메인 네비게이션 (Admin에서)
- **푸터**: 버전, 링크

### 4.2 재사용 컴포넌트
- 버튼 (Primary/Secondary/Danger)
- 모달/다이얼로그
- 테이블 (정렬, 페이지네이션)
- 카드
- 폼 요소 (Input, Select, Checkbox)
- 차트 (레이더, 라인, 바)
- 배지/태그
- 로딩 스피너
- 토스트 알림

## 5. 디자인 시스템 권장사항

### 5.1 컬러 팔레트
- **Primary**: 파랑 계열 (과학/의료 이미지)
- **Secondary**: 보라 계열
- **Success**: 초록
- **Warning**: 노랑
- **Danger**: 빨강
- **Neutral**: 그레이 스케일

### 5.2 타이포그래피
- 제목: 산세리프 (Inter, Pretendard)
- 본문: 가독성 높은 폰트
- 코드/데이터: 고정폭 폰트

### 5.3 다크 모드
- 기본 다크 테마 권장 (과학 도구 특성)
- 라이트 모드 토글 옵션
