2. Problem & Solution: 왜 이 플랫폼인가?
2-1. 연구원님의 고민 (Problem)
ADC 개발, 레고 블록 맞추기보다 수천 배는 어렵습니다.

너무 많은 경우의 수: 항체, 링커, 페이로드를 조합하면 수십만 가지가 넘습니다. 어디서부터 시작해야 할까요?

숨겨진 데이터: 최신 약물의 구조와 실패 원인은 논문과 특허 속에 흩어져 있거나 숨겨져 있습니다.

값비싼 시행착오: 합성하고 테스트하는 데 드는 시간과 비용, 미리 줄일 수 없을까요?

설명: 수많은 후보 물질 중 극소수만 성공하는 '깔때기(Funnel)' 이미지를 통해, 초기 선별의 중요성을 시각화합니다.

2-2. 우리의 해답 (Solution)
우리는 '가능성'을 설계하는 엔진입니다. 막연한 기대가 아닌, 데이터와 화학적 계산으로 검증합니다. 이미 성공했거나 실패한 사례(Golden Set)를 학습하여, 귀하가 설계한 조합이 '상업적으로 가치가 있는지', '화학적으로 안정적인지' 점수로 환산해 드립니다.

3. Key Features: 핵심 기능 6가지
1) 🎯 성공 확률 예측 (AI Scoring)
"이 조합, 점수로 보면 몇 점일까?" 표적과 항체만 입력하세요. 최적의 링커와 페이로드를 추천하고, 이 조합의 **개발 적합성(표적 적합도, 링커 안정성 등)**을 0~100점 척도로 보여드립니다.

2) 🔍 근거 기반 탐색 (Traceable Evidence)
"AI의 환각 없는, 팩트 기반 결과" "좋다"고 말만 하지 않습니다. "어떤 임상 시험에서 효과가 있었는지", "어떤 특허에 나온 구조인지" 출처 링크를 100% 제공합니다. 믿을 수 있는 정보만 드립니다.

3) 🧪 화학적 리스크 분석 (Chemical Safety Check)
"만들기 전에 위험을 감지합니다" 구조식만으로도 알 수 있습니다. RDKit 기술을 활용해 물에 잘 녹는지, 너무 잘 뭉치지는 않는지(응집성), 실제 약으로 만들 때 문제는 없을지 미리 계산해 드립니다.

설명: 분자 구조식 위로 분석 데이터가 오버레이 되는 이미지를 통해, 정밀한 화학 분석 기능을 강조합니다.

4) 📝 연구 프로토콜 가이드 (Protocol Assistant)
"다음 실험, 무엇을 해야 할지 알려줍니다" 선택한 후보 물질을 검증하기 위해 당장 필요한 실험이 무엇인지(안정성 테스트, 독성 평가 등), 체크리스트를 자동으로 생성해 드립니다.

5) 🏆 정예 데이터 학습 (Golden Set Curation)
"쓰레기 데이터는 넣지 않았습니다" 자동으로 긁어모은 노이즈 데이터가 아닙니다. 전문가가 직접 검수하고 승인한 **'고밀도 정답 데이터(Golden Set)'**만을 학습하여 추천의 정확도를 높였습니다.

6) 📄 원클릭 보고서 (Auto Reporting)
"회의 자료, 3초 만에 완성" 시뮬레이션 결과, 근거 자료, 화학적 분석 수치, 그리고 예상되는 리스크까지. **한 편의 완벽한 기술 검토 보고서(PDF)**로 만들어 드립니다.

4. How it Works: 작동 방식
복잡한 과정은 저희가 처리합니다. 연구에만 집중하세요.

입력 (Input): 타겟(Target)과 암종, 혹은 선호하는 항체를 입력합니다.

탐색 (Discovery): 전 세계 임상 데이터와 화학 DB를 뒤져 최적의 후보(링커/페이로드)를 찾습니다.

검증 (Validation): 구조가 없는 최신 약물은 '유사 구조(Proxy)'를 통해 빈틈없이 시뮬레이션합니다.

결과 (Result): 점수화된 후보 리스트와 상세 분석 보고서를 확인합니다.

5. Golden Set & Integrity: 신뢰의 기준
품질에 타협은 없습니다 (Quality First)
우리는 데이터를 무작위로 수집하지 않습니다.

엄격한 관리: 관리자(Admin)가 직접 검수한 데이터만 시스템의 기준(Seed)이 됩니다.

투명한 공개: 구조가 100% 확실하지 않은 최신 약물의 경우, **"유사 구조(Proxy) 기반 추정값"**임을 투명하게 명시합니다.

성공과 실패의 균형: 성공 사례뿐만 아니라, **임상에서 실패한 사례(독성, 효능 부족)**까지 학습하여 리스크를 피해 가도록 돕습니다.

7. FAQ (자주 묻는 질문)
Q. PubMed 검색이랑 뭐가 다른가요? A. PubMed는 '글'을 찾아주지만, 우리는 '데이터'를 분석합니다. 논문의 텍스트를 넘어, 실제 화학 구조와 임상 수치를 연결하여 시뮬레이션 결과를 제공한다는 점이 다릅니다.

Q. 최신 약물(예: MK-2870) 정보도 있나요? A. 네, 있습니다. 아직 구조가 완전히 공개되지 않은 약물은 임상 데이터와 유사 구조(Proxy) 기술을 활용해 최대한 정밀하게 분석하여 정보를 제공합니다.

Q. 이 결과만 믿고 임상을 진행해도 되나요? A. 본 플랫폼은 연구 성공 확률을 높여주는 '설계 및 가이드' 도구입니다. 최종적인 비임상/임상 실험 검증은 연구 기관의 몫이지만, 우리는 그 과정의 시행착오를 획기적으로 줄여드립니다.
