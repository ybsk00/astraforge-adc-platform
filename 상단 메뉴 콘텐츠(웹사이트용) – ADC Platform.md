# 상단 메뉴 콘텐츠(웹사이트용) – ADC Platform

본 문서는 상단 메뉴 **서비스 소개 / 기능 / 솔루션**에 들어갈 상세 텍스트 초안입니다.  
(홈페이지 톤: B2B, 기술 신뢰 + 도입 효율 강조)

## 1) 서비스 소개

### ADC Platform이란?
ADC Platform은 항체-약물 접합체(ADC) 연구개발 과정에서 발생하는 **표적 탐색 → 후보 설계 → 근거 검증 → 우선순위 결정**을 하나의 워크플로우로 연결하는 **AI 기반 Drug Discovery 플랫폼**입니다.  
논문·특허·공개 DB·사내 데이터 등 분산된 정보를 자동으로 수집·정리하고, 연구자가 원하는 질문에 대해 **근거 기반으로 빠르게 답변**하며, 후보 조합의 성공 가능성을 정량적으로 비교할 수 있도록 지원합니다.

### 해결하는 문제
- 데이터가 흩어져 있어 **표적/항체/링커/페이로드 후보를 비교**하기 어렵습니다.
- 근거를 찾고 정리하는 데 시간이 과도하게 소요되어 **실험 설계가 느려집니다.**
- ADC 설계는 변수(표적, 항체, 링커, 페이로드, DAR, 안정성 등)가 많아 **의사결정이 경험 의존**이 되기 쉽습니다.
- 팀 간 문서/결과물 관리가 분산되어 **재현성과 협업 효율이 저하**됩니다.

### 플랫폼이 제공하는 핵심 가치
- **Research Time 단축:** 검색·정리·리포트 자동화로 연구 리드타임을 줄입니다.
- **Evidence-First(근거 우선):** 답변과 추천이 항상 출처/근거와 함께 제공되어 검증이 쉽습니다.
- **Design-to-Decision:** 설계안 생성부터 비교/스코어링/우선순위까지 연결합니다.
- **조직형 협업:** 프로젝트/권한/감사로그 기반으로 연구 조직의 협업을 표준화합니다.

### 사용 흐름(예시)
1. **표적/적응증 정의:** 암종·바이오마커·기전 가설 입력  
2. **데이터 수집 및 정리:** 논문/특허/DB 자동 수집 + 지식화(청킹/인덱싱)  
3. **후보 설계:** 항체·링커·페이로드 조합 후보 생성 및 후보군 정리  
4. **근거 기반 검증:** RAG 기반 질의응답으로 근거 문장/논문/특허 확인  
5. **스코어링/우선순위:** 성공 가능성·리스크·개발성 관점의 정량 비교  
6. **리포트/공유:** 투자·내부 심의·실험 계획을 위한 결과물 자동 생성

### 차별점
- **Hybrid Search(키워드+의미 검색):** 중요한 키워드를 놓치지 않으면서도 문맥 기반 탐색이 가능합니다.
- **Explainable Recommendation:** “왜 이 후보가 유리한지”를 근거와 함께 제시합니다.
- **모듈형 확장:** 데이터 커넥터/모델/스코어링 규칙을 조직 목적에 맞게 확장합니다.
- **엔터프라이즈 운영:** 권한·보안·감사·배포 옵션을 고려한 운영형 구조를 제공합니다.

## 2) 기능

### 2.1 데이터 수집(Connectors) & 지식화
- **외부 데이터 커넥터**
  - PubMed/NCBI 기반 논문 수집
  - 특허/공개 데이터(조직 정책에 따라 확장)
  - 표적·단백질·경로·질환 관련 공개 DB 연동(옵션)
- **사내 데이터 연동(옵션)**
  - 연구노트/리포트/PDF/스프레드시트 업로드
  - 사내 표준 템플릿 기반 메타데이터 정규화
- **지식화 파이프라인**
  - 문서 구조 분석(섹션/문단/문장 단위)
  - 청킹(Chunking) + 임베딩(Embedding) + 인덱싱
  - 중복 제거, 버전 관리, 출처(Reference) 보존

### 2.2 Hybrid Search & RAG 질의응답
- **키워드 검색(BM25) + 의미 검색(Vector) 결합**
  - 정확 매칭과 의미 유사도를 동시에 고려
  - 최신성/신뢰도/저널/연도 필터 등 정교한 검색 옵션
- **근거 기반 답변(RAG)**
  - 답변에 사용된 근거 문장/출처를 함께 제공
  - 주장-근거 매핑으로 검증 용이
  - 동일 질문에 대한 재현성 있는 결과 제공(세션 기록)

### 2.3 ADC 설계 지원(Design Workspace)
- **설계 요소 관리**
  - 표적(Target), 항체(Antibody), 링커(Linker), 페이로드(Payload), DAR 등 핵심 파라미터 구조화
  - 후보 조합(ADC construct) 생성 및 버전 관리
- **후보 비교/정리**
  - 후보군 리스트업, 태깅, 노트, 코멘트, 상태 관리
  - 팀 단위 리뷰를 위한 비교 테이블 및 요약 카드 제공

### 2.4 스코어링 & 리스크 평가(의사결정 지원)
- **성공 가능성 관점의 정량 비교(옵션)**
  - 표적 적합성/발현 특성/내재화 가능성/선행연구 근거
  - 링커 안정성/방출 메커니즘 적합성
  - 페이로드 potency/독성 위험/내성 가능성
  - 개발성(Developability) 및 제조·품질(CMC) 관점 리스크 체크
- **설명 가능한 스코어링**
  - 점수의 근거와 기여 항목을 함께 제공(“왜 점수가 높은가/낮은가”)

### 2.5 협업 & 운영(Enterprise Ready)
- **프로젝트/워크스페이스**
  - 팀/과제 단위 분리, 템플릿 기반 운영
- **권한/보안**
  - 사용자·역할 기반 접근 제어(RBAC)
  - 감사 로그(Audit Log), 데이터 접근 이력 관리
- **리포트 생성**
  - 내부 심의/투자/실험계획용 요약 리포트 자동 생성
  - PDF/문서 내보내기(옵션)
- **API/확장성(옵션)**
  - 외부 시스템(LIMS/ELN/스토리지) 연동
  - 커넥터/룰/모델 플러그인 방식 확장

## 3) 솔루션

### 3.1 타겟 인텔리전스(Target Intelligence)
- 암종/적응증 기반으로 **유망 표적 후보군**을 도출하고,
- 표적별 근거(논문/특허/기전/임상 동향)를 구조화하여
- “지금 개발할 가치가 있는 표적인가?”를 빠르게 판단할 수 있도록 지원합니다.

**적용 예**
- 신규 표적 탐색, 경쟁 파이프라인 조사, 적응증 확장 가능성 검토

---

### 3.2 ADC 설계 최적화(ADC Design Optimization)
- 항체·링커·페이로드 조합을 체계적으로 정리하고,
- 조합별 리스크 포인트를 조기에 확인하여,
- **실험 우선순위를 데이터 기반으로 결정**할 수 있도록 지원합니다.

**적용 예**
- 링커 안정성 가정 비교, 페이로드 선택 근거 정리, 후보군 압축(Down-selection)

---

### 3.3 개발성/안전성 리스크 점검(Developability & Safety)
- 후보 물질의 **개발 관점 리스크(안정성/응집/면역원성 가능성 등)**를 점검하고,
- 안전성 우려 신호를 근거와 함께 제시하여,
- 초기 단계에서의 실패 비용을 줄이는 데 집중합니다.

**적용 예**
- “실험하기 전에 위험한 후보를 거르는” 스크리닝 단계 구축

---

### 3.4 연구 조직형 지식관리(Knowledge Ops)
- 개인 PC/폴더/메일/메신저에 흩어진 자료를
- 프로젝트 단위로 수집·정리하여 **조직 지식 자산**으로 전환합니다.

**적용 예**
- 신규 인력 온보딩, 과제 인수인계, 내부 표준 리포트 템플릿 운영

---

### 3.5 도입 형태(옵션)
- **MVP(빠른 도입):** 핵심 커넥터 + 검색/RAG + 기본 워크스페이스
- **팀 확장:** 스코어링/리포트 자동화/협업 기능 강화
- **엔터프라이즈:** SSO, 감사로그, 전용 배포(VPC/On-prem), SLA, 커스텀 커넥터
